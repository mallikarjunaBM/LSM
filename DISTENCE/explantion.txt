The program is designed to estimate the movement of an **LSM303DLHC sensor** connected to a **Raspberry Pi Pico** through the I2C communication protocol. When the program starts, it first imports the required MicroPython libraries such as `machine` for hardware access, `utime` for timing operations, and `math` for mathematical calculations. The I2C interface is configured using **GP0 as SDA** and **GP1 as SCL**, and the addresses of both the accelerometer (`0x19`) and magnetometer (`0x1E`) are defined. The `sensor_init()` function initializes the accelerometer and magnetometer by writing configuration values into their control registers so that both sensors begin producing measurement data. The `write_reg()` and `read_reg()` functions perform reliable I2C communication with retry mechanisms to reduce the chance of communication failures such as `OSError` or `EIO`. The `signed12()` and `signed16()` functions convert the raw sensor data into signed integers because the sensor stores measurements in two's complement format. The `read_acc()` function continuously reads the raw X, Y, and Z acceleration values from the accelerometer, while the `read_mag()` function reads the magnetic field values from the magnetometer. Before normal operation begins, the `calibrate()` function collects several hundred samples while the sensor remains stationary and calculates the average sensor bias. These bias values are later removed from every measurement inside the `get_acc()` function, producing calibrated acceleration values. The program converts these calibrated readings into acceleration values expressed in metres per second squared (m/s²). The `calculate_angles()` function computes the **roll** and **pitch** angles using trigonometric equations based on the three acceleration components, while the `calculate_heading()` function combines the magnetometer measurements with the roll and pitch values to calculate a **tilt-compensated heading**, allowing the compass direction to remain accurate even when the sensor is tilted. The `filter_values()` function applies a low-pass filter to smooth the roll, pitch, and heading values by reducing the effects of sensor noise and sudden fluctuations. The `gravity_compensation()` function removes the Earth's gravitational acceleration from the measured acceleration so that only the actual linear motion of the sensor remains. The `estimate_motion()` function performs numerical integration by first calculating the current velocity from the linear acceleration and then integrating the velocity again to estimate the sensor's displacement. It also computes the overall estimated distance travelled from the X, Y, and Z position components. The `direction()` function converts the heading angle into human-readable compass directions such as North, East, South, West, and the intermediate directions. Finally, the `display()` function prints the roll, pitch, heading, compass direction, velocity, and estimated distance to the Thonny Shell in real time. The `main()` function controls the complete execution of the program by initializing the sensor, performing calibration, repeatedly reading sensor data, calculating orientation and motion parameters, estimating the travelled distance, handling communication errors, and continuously displaying the updated results until the program is stopped.




The logic of the program begins by establishing communication between the Raspberry Pi Pico and the LSM303DLHC sensor using the I2C protocol. After the sensor is detected, the accelerometer and magnetometer are configured by writing the required values into their control registers. The program then performs a calibration process while the sensor is kept stationary. During calibration, several hundred samples of the accelerometer are collected and averaged to determine the sensor bias, which is later removed from every measurement to improve accuracy. After calibration, the program enters an infinite loop where it continuously reads the X, Y, and Z acceleration values from the accelerometer and the X, Y, and Z magnetic field values from the magnetometer. The raw sensor values are converted into signed integers and then into acceleration values expressed in metres per second squared. Using the calibrated acceleration values, the program calculates the **roll** and **pitch** angles to determine the sensor's orientation. Next, the magnetometer readings are combined with the roll and pitch angles to calculate a **tilt-compensated heading**, ensuring that the compass direction remains correct even when the sensor is tilted. A low-pass filter is then applied to the roll, pitch, and heading values to remove sensor noise and produce stable output. The program performs gravity compensation by removing the Earth's gravitational acceleration from the measured acceleration, leaving only the actual linear acceleration caused by sensor movement. This linear acceleration is integrated over time to calculate the current velocity along the X, Y, and Z axes. The calculated velocity is then integrated again to estimate the sensor's position in three dimensions. Finally, the program computes the overall estimated distance travelled by calculating the magnitude of the X, Y, and Z position values. The heading angle is also converted into a compass direction such as North, East, South, or West for easier understanding. All the calculated values, including roll, pitch, heading, compass direction, velocity, and estimated distance, are displayed continuously in the Thonny Shell. The entire process repeats continuously until the user stops the program, while built-in error handling ensures that temporary I2C communication errors do not terminate the application.
